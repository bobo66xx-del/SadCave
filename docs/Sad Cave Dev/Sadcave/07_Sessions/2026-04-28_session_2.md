## 🤔 Decisions

Six decisions captured in `_Decisions.md` this session (all 2026-04-28). Three from session 7's design walk that hadn't been backfilled, two new from MVP-2's brief drafting (via AskUserQuestion), and one new workflow rule:

- **Title-on-nametag stays.** Sanity check after PR #9's level-row strip. Title row went back at BillboardGui height 50 with NameLabel + TitleLabel.
- **TitleService becomes a ModuleScript.** PR #13's architecture cleanup. Replaces the `_G.SadCaveTitleService` shared-state pattern from PR #12 with a direct `require`. Loud-failure mode preferred over silent-fallback.
- **Leaderstats persistence confirmed; level watcher re-attach kept as cheap insurance.** PR #13's probe found `leaderstatsSame=true` across respawns, so the original concern was theoretical. Re-attach stays anyway — one line, idempotent, self-heals if a future ProgressionService change ever swapped leaderstats.
- **Manual equip respects player choice forever.** MVP-2's gating rule. Auto-equip-highest becomes first-time-only fallback. Unlock fades still fire so players know about new options.
- **Menu entry point: small lowercase 'titles' text button in the top-right corner.** MVP-2's UX choice. Lowest UI surface, easy to remove when the polished menu lands, doesn't fragment platform UX.
- **Workflow: Claude integration sweeps land via `claude/integration-...` branch + user merge.** AGENTS.md gains step 10 in the End-of-Session bookend. Closes the vault-revert pattern by construction — once vault commits land on main, no subsequent local activity can revert them.
