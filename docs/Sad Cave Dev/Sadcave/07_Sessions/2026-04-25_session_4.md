## 🤔 Decisions

- **Sitting > exploring > AFK for XP rates.** User called this out specifically: a hangout game should reward chilling at a bench more than walking around. Original spec had sitting as a small flat bonus on top of presence tick; redesigned to be a full state replacement (boosted tick rate while seated 30s+).
- **Combined DataStore keys.** Originally spec'd four separate keys (`XPSave`, `DiscoveredZones`, `Achievements_v1`, `EquippedTitleV2`). Consolidated to two (`ProgressionData`, `TitleData`) after explaining DataStore request limits. Two reads on join instead of four.
- **MVP cuts deeply.** Discovery/Conversation/Achievement/Title v2 all moved to follow-up briefs. The MVP just replaces `LevelLeaderstats` with the new ProgressionService and adds the XP bar. Smaller scope = safer first ship, and proves the architecture before layering more on it.
- **Feature-flag cutover pattern.** `SourceConfig.ENABLED = false` ships in the committed code. Codex flips it locally for testing, then in production Studio for the cutover. Means the code can land in main without touching live behavior, and rollback is a one-line edit.
- **Title naming convention: lowercase with spaces.** Reads like a whispered thought, not a code variable, not a formal rank. Matches dialogue voice ("you came back", "it's quieter when you're here").
- **Three name changes locked in:** `glass hour` → `gone quiet` (level 2000), `permanent resident` → `always was here` (level 10000), `neon haze` → `soft static` (gamepass).
- **`up_too_late` uses client-reported timezone.** Server gets UTC offset from client on join. Low security risk for a cosmetic title — worst case a player lies and earns a title they didn't.
- **Gamepass titles all under existing ID `1797105034`.** One purchase unlocks all six. No new gamepasses needed.

## 📝 Notes

- **First pure-design session.** No Studio inspection, no Codex execution, no reconciliation. Just the user, Opus, and the vault. Worked well — design surface stays clean when there's no code work happening in parallel.
- **The `ask_user_input_v0` tool was used heavily** to make decisions efficient — multi-question gut-check rounds let the user pick from concrete options instead of writing prose answers. Especially useful for naming and pacing-target decisions.
- **Numerical balancing was an iterative search.** Brute-force optimizer couldn't satisfy all six pacing targets with a smooth single-formula curve (early levels needed to be very fast, late levels very slow — incompatible with a single exponent). Solution was to accept the optimizer's best-fit curve and *reframe the title placement* around where the curve naturally lands, rather than forcing the curve to match arbitrary title thresholds. Title and XP designs ended up shaped by each other.
- **The "permanent resident" → "always was here" change is the kind of tonal call that's hard to spot in spec but obvious in retrospect.** v1 had this energy throughout — `superstar`, `legend`, `divine` — and v2 systematically replaces it with internal/atmospheric language. Worth watching for the same drift if more titles get added in the future.
- **MVP plan size is right.** Long enough to be self-contained (Codex shouldn't need to ask follow-up questions for the MVP scope); short enough that follow-up briefs don't have to repeat its content. Studio Test Checklist is the most important section — it's the gate between Phase 6 testing and Phase 7 production cutover.
