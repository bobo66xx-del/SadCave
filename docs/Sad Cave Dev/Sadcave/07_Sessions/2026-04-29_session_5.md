## 🐛 Broke / Fixed

- **`up_too_late` achievement bug root-caused.** Tyler runtime-tested at 10:59pm Central time and the title fired wrongly (spec is 3am-5am local time only). Bug is in the client UTC offset computation: `os.difftime(os.time(), os.time(os.date("!*t", now)))` returns 0 in Roblox because Roblox's `os` library is UTC-only — both `os.time()` and `os.time(os.date("!*t", now))` operate on UTC, the difference is always zero. Client sends offset=0, server checks UTC hour, 10:59pm CDT = 03:59 UTC, satisfies `hour == 3`, flag fires. Bug is silent for players near UTC and incorrect for everyone else. Filed in `_Known_Bugs.md` as new active bug. Fix is one function (~5 lines): use `os.time(os.date("*t")) - os.time(os.date("!*t"))` since the unprefixed `*t` returns local time on the **client**. Can ship as a tiny one-file PR (`codex/up-too-late-offset-fix`) any time. Bug originated in PR #20 (AchievementTracker, 2026-04-29 08:52 UTC); good runtime-vs-static-review surfacing — formula reads correct in standard Lua but wrong in Roblox's UTC-only `os` library context.
