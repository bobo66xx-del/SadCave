## 📝 Notes

- **First "full loop in one session" since the live-build period started.** Design → brief → Codex → review → integration all happened end-to-end without a session boundary in between. Worked because the brief was scoped to a single Codex sitting (~5 hours from brief-write to Claude review-pass) and because Rojo got reconnected mid-session so the playtest could actually run. When Rojo was disconnected (pre-reconnect), Codex did a clean push without runtime verification — a reminder that playtest-blocked status is a common-enough failure mode that the review template needs to handle it gracefully (it does — verdict was "hold" until playtest available, then upgraded to "safe to merge" after).
- **The min-NPC gate pattern is reusable.** Any achievement that depends on a roster (NPCs, zones, items) can use this shape: persist the per-player "covered set" + check against a `MIN_X` constant. Cheap future-proofing, no migration when the roster grows. Worth keeping in mind for future achievement design.
- **Capture-but-don't-surface from QuietKeeper_Memory's MVP made an appearance here too.** AchievementTracker stores `npcsHeard` per player from day one even though `heard_them_all` is gated dormant — when the gate opens later, every player's existing visit history counts. Same pattern: data shape stability ahead of surface readiness.
- **Auto-equip priority change worth flagging:** achievement titles with effects (`shimmer`/`pulse`/`glow`) can outrank lower-level titles in auto-equip-highest. A player at level 50 who unlocks `part_of_the_walls` (shimmer, score 100) would auto-swap to that achievement title. Tyler's manually-equipped `familiar_face` is being respected because of `equippedManually = true` from his prior manual pick. Behavior is intentional — earned loyalty markers should compete with level milestones — but worth knowing in case Tyler sees auto-swap behavior he didn't expect.
