## ✅ Built / Shipped

- **`01_Vision/Player_Experience_Arcs.md` — new vision doc.** Sibling to `Tone_and_Rules` and `Project_Overview`. Names three horizons (arrival / return / loyalty) + what the player feels in each + what currently happens + where each thins. Diagnosis: days 3–7 of the return arc are the weakest stretch (low design density relative to retention weight, leverage on month-3 conversion, actionability via inactive systems already specced). Includes a "How to use this doc" section so future system proposals can be evaluated against it ("which horizon is this for? what does it make the player feel?").
- **`02_Systems/QuietKeeper_Memory.md` — new spec at 🔵 Planned.** Loyalty-arc centerpiece. Tracks 8 fields per player (visit count, last visit, current/longest streaks, total seated seconds, first visit, recent unlocks FIFO, hasBeenOutside flag); MVP gates on visit count only with the other 7 fields captured-but-dormant to avoid future migration. Surfaces as conditional dialogue line variants extending the existing `return` line — the existing line stays as the fallback. Seven binding tone rules baked into the spec (under 10 words, no numbers, no celebration, default-line-stays-live, higher-tier-can-be-quieter, silence-allowed, no-4th-wall). Dependency: `Dialogue_System` extension to support conditional variant lists is the build-blocker for the MVP brief.
- **Three loyalty-arc stub specs at ⚪ Idea:** `Personal_Place.md` (cave noticing your favorite sit-spot), `Seasonal_Layer.md` (calendar-aware shifts in environment + NPC moods + Seasonal title windows), `Veteran_Threshold_Content.md` (beats only visible after long playtime / return-count). Each ~50-line stub includes idea, rationale, what it touches, open questions, dependencies, "designed when" trigger, risks, related links.
- **`02_Systems/Title_System.md` Category Activation Sequence rewrite.** Replaced the system-readiness-ordered "Follow-up briefs" section with a player-experience-anchored sequence: AchievementTracker first (densest beats in days 3–7), Presence second (cheap counter, reinforces gap), Exploration third (paired with Outside + Area_Discovery), Seasonal last (gated on Seasonal_Layer). Polished TitleMenu and v1 retirement repositioned as parallel Tyler-led tracks.
- **`00_Index.md` updates.** Added Player_Experience_Arcs under Vision quick links; added QuietKeeper_Memory + the three stubs under Active Systems with statuses; annotated Title_System with a pointer to the new Category Activation Sequence section; expanded the existing "Polished TitleMenu" Active focus item to "Polished TitleMenu + nametag title visual" with framing for the nametag-visual scope per Tyler's earlier remark.
