## Format

`YYYY-MM-DD HH:MM UTC — [System] — Change. Reason.`

Use `HH:MM UTC` when the timestamp is known. Use `(<session label>)` (e.g. `(Cowork session 1, mid)`, `(pre-Cowork morning)`) for entries scoped only to a session. **Sort newest on top by event time, not by when the entry was written** — out-of-session work (Codex executing briefs, automated merges) often happens between sessions and slots into the timeline above older session-1 entries.

Examples:
`2026-04-27 13:32 UTC — Repo — PR #5 merged. Restored DialogueDirector. Closes the dialogue gap from the cleanup pass.`
`2026-04-27 (Cowork session 1, mid) — 02_Systems — Per-system status reframing post-cleanup.`
`2026-04-25 — Dialogue_System — Bumped default cooldown 3s → 4s. Felt rushed in playtest.`
